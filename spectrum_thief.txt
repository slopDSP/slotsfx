No, the algorithm for Ghost Note Audio's Spectrum Thief is not open source.While Spectrum Thief is distributed as a FREE plugin (meaning you can download, use, and share it without paying, registering, or authenticating), it is closed-source freeware. The creator, Ghost Note Audio, does not make the underlying C++ code, JUCE source files, or the exact math behind their EQ-matching and Impulse Response (IR) generation logic publicly available.How to reproduce that exact algorithm anywayEven though Spectrum Thief's exact code is locked up, the mathematical process of EQ-matching and IR generation is entirely standardized, well-documented, and widely available in open-source formats.If you want to write or wire up your own version of this plugin, you are looking for a Linear Deconvolution / Transfer Function algorithm. It works via the following open-source concepts:The Core Math (FFT / Fast Fourier Transform)To match a "Source" to a "Target", you convert both audio blocks from the time domain to the frequency domain using an FFT algorithm (like fftw or JUCE’s juce::dsp::FFT).$$\text{Source Spectrum} = \text{FFT}(\text{Source Audio})$$$$\text{Target Spectrum} = \text{FFT}(\text{Target Audio})$$The EQ Match Calculation (Complex Division)The adjustment filter (the "Thief" layer) is calculated by dividing the target's frequency spectrum by the source's frequency spectrum.$$\text{Matching Filter Spectrum} = \frac{\text{Target Spectrum}}{\text{Source Spectrum}}$$(Note: Open-source algorithms add a small smoothing value—called a waterfilling parameter—to the denominator to prevent dividing by zero where the audio is totally silent).Generating the IR (IFFT)Once you have the Matching Filter Spectrum, you convert it back into standard audio samples using an Inverse Fast Fourier Transform (IFFT). Those resulting samples are the literal Impulse Response file (.wav) that you can export or run through a standard convolution engine (like the one in NAM or JUCE's juce::dsp::Convolution).Open-Source Alternatives to ReferenceIf you want to look at working, open-source repositories that do exactly what Spectrum Thief does under the hood, look up:Python implementations: Look for open-source repositories utilizing scipy.signal.deconvolve or Transfer Function estimation (scipy.signal.tfestimate). These libraries take two audio files, find the difference, and export a .wav impulse response.C++ implementations: Look up open-source JUCE plugins that utilize juce::dsp::FFT paired with a threshold gate to calculate frequency differentials between two incoming channels.
